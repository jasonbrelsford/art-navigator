Data Sources

Add custom JSON endpoints or URLs that return entity data. The graph will fetch and merge results from all enabled sources.

URL must contain {query} as placeholder. Response should be JSON with an array of objects containing at minimum: label, description.

Built-in: Wikidata (always enabled)

Host Your Own Graph

1

Fork the repository

Go to the GitHub repo and click Fork in the top right. This copies the entire app to your GitHub account.

2

Enable GitHub Pages

In your forked repo, go to Settings → Pages. Under "Build and deployment", select GitHub Actions as the source. The included workflow will deploy automatically.

3

Visit your site

After a minute or two, your graph will be live at:
https://YOUR-USERNAME.github.io/art-navigator/

4

Start exploring

Search for anything — artists, musicians, politicians, companies, places. Your graph data saves to your browser's localStorage automatically. Use Export/Import to back up or share your graph.

How data works

Your graph lives in your browser (localStorage). It's private to you and persists across sessions. The app itself is static HTML/JS — no server, no database, no accounts. You own everything.

To share a specific graph with someone, click ↓ Export to download a JSON file, then they can ↑ Import it into their own instance.
